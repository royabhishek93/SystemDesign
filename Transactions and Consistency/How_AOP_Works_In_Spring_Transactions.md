## What is AOP in Spring Transactions

### Simple Explanation

**AOP (Aspect-Oriented Programming)** allows you to add behavior (like transaction management) to methods **without modifying the method code**.

Think of it like a wrapper:

```
┌─────────────────────────────────────────────────────────┐
│                   WITHOUT AOP                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  public void saveUser(User user) {                     │
│      // YOU have to write:                             │
│      TransactionStatus tx = txManager.getTransaction();│
│      try {                                              │
│          userRepository.save(user);                     │
│          txManager.commit(tx);                          │
│      } catch (Exception e) {                            │
│          txManager.rollback(tx);                        │
│          throw e;                                       │
│      }                                                  │
│  }                                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    WITH AOP                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  @Transactional                                         │
│  public void saveUser(User user) {                     │
│      userRepository.save(user);  // Simple!            │
│  }                                                      │
│                                                         │
│  AOP automatically adds transaction management          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## How @Transactional Works Internally

### Step-by-Step Flow

```
┌─────────────────────────────────────────────────────────┐
│         HOW @TRANSACTIONAL WORKS (AOP MAGIC)            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Step 1: Spring Startup                                │
│  ───────────────────────────────────────────────────   │
│                                                         │
│  Spring scans your code:                               │
│  "UserService has @Transactional methods!"            │
│                                                         │
│  ┌─────────────────────────────────┐                  │
│  │  @Service                        │                  │
│  │  public class UserService {      │                  │
│  │      @Transactional              │ ← Found!         │
│  │      public void saveUser() {}   │                  │
│  │  }                               │                  │
│  └─────────────────────────────────┘                  │
│                                                         │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                                         │
│  Step 2: Create Proxy                                  │
│  ───────────────────────────────────────────────────   │
│                                                         │
│  Spring creates a PROXY wrapper:                       │
│                                                         │
│  ┌─────────────────────────────────────────┐          │
│  │  UserServiceProxy (generated by Spring) │          │
│  │  ──────────────────────────────────────  │          │
│  │                                          │          │
│  │  public void saveUser(User user) {      │          │
│  │      // BEFORE: Start transaction       │          │
│  │      txManager.begin();                 │          │
│  │                                          │          │
│  │      try {                               │          │
│  │          // Call REAL method            │          │
│  │          realUserService.saveUser(user);│          │
│  │                                          │          │
│  │          // AFTER: Commit               │          │
│  │          txManager.commit();            │          │
│  │      } catch (Exception e) {            │          │
│  │          // ERROR: Rollback             │          │
│  │          txManager.rollback();          │          │
│  │          throw e;                       │          │
│  │      }                                   │          │
│  │  }                                       │          │
│  └─────────────────────────────────────────┘          │
│                                                         │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                                         │
│  Step 3: Inject Proxy (Not Real Object)               │
│  ───────────────────────────────────────────────────   │
│                                                         │
│  When you @Autowire UserService:                      │
│                                                         │
│  @Controller                                            │
│  public class UserController {                         │
│      @Autowired                                         │
│      private UserService userService;  ← This is PROXY!│
│  }                                                      │
│                                                         │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                                         │
│  Step 4: Method Call Flow                             │
│  ───────────────────────────────────────────────────   │
│                                                         │
│  userService.saveUser(user);                           │
│         │                                               │
│         ▼                                               │
│  ┌──────────────┐                                      │
│  │ PROXY Object │  (Spring created)                    │
│  └──────┬───────┘                                      │
│         │                                               │
│         │ 1. Begin Transaction                         │
│         │ 2. Call real method                          │
│         │ 3. Commit/Rollback                           │
│         │                                               │
│         ▼                                               │
│  ┌──────────────┐                                      │
│  │ REAL Object  │  (Your code)                         │
│  └──────────────┘                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Proxy-Based Architecture

### Visual Architecture

```
┌──────────────────────────────────────────────────────────┐
│            SPRING PROXY ARCHITECTURE                     │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌────────────────────────────────────────────────┐    │
│  │         CLIENT (Controller)                    │    │
│  │                                                │    │
│  │  @Autowired                                    │    │
│  │  private UserService userService;              │    │
│  │                                                │    │
│  │  userService.saveUser(user);  ← Calls proxy   │    │
│  └────────────────┬───────────────────────────────┘    │
│                   │                                     │
│                   ▼                                     │
│  ┌────────────────────────────────────────────────┐    │
│  │   UserService$$SpringProxy (CGLIB Proxy)       │    │
│  │   ────────────────────────────────────────     │    │
│  │                                                │    │
│  │   public void saveUser(User user) {           │    │
│  │                                                │    │
│  │     // ────── AOP ADVICE (Before) ──────      │    │
│  │     TransactionInterceptor.invoke() {         │    │
│  │         txManager.getTransaction();           │    │
│  │         Connection conn = getConnection();    │    │
│  │         conn.setAutoCommit(false);            │    │
│  │                                                │    │
│  │     // ────── CALL REAL METHOD ──────         │    │
│  │         targetObject.saveUser(user);          │    │
│  │                          │                     │    │
│  │                          ▼                     │    │
│  │     // ────── AOP ADVICE (After) ──────       │    │
│  │         if (success) {                         │    │
│  │             conn.commit();                     │    │
│  │             txManager.commit();                │    │
│  │         } else {                               │    │
│  │             conn.rollback();                   │    │
│  │             txManager.rollback();              │    │
│  │         }                                      │    │
│  │     }                                          │    │
│  │   }                                            │    │
│  └────────────────┬───────────────────────────────┘    │
│                   │                                     │
│                   ▼                                     │
│  ┌────────────────────────────────────────────────┐    │
│  │   UserService (REAL/TARGET Object)             │    │
│  │   ─────────────────────────────────            │    │
│  │                                                │    │
│  │   @Transactional                               │    │
│  │   public void saveUser(User user) {           │    │
│  │       userRepository.save(user);  ← Your code │    │
│  │   }                                            │    │
│  └────────────────────────────────────────────────┘    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### Two Types of Proxies

Spring uses two types of proxies:

```
┌──────────────────────────────────────────────────────────┐
│              PROXY TYPES IN SPRING                       │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  1. JDK Dynamic Proxy                                   │
│  ──────────────────────────────────────────────────     │
│                                                          │
│  When: Class implements interface                       │
│                                                          │
│  interface UserService {                                │
│      void saveUser(User user);                          │
│  }                                                       │
│                                                          │
│  @Service                                                │
│  class UserServiceImpl implements UserService {         │
│      @Transactional                                      │
│      public void saveUser(User user) { ... }            │
│  }                                                       │
│                                                          │
│  → Spring creates: UserService$$JdkProxy                │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                                          │
│  2. CGLIB Proxy (More Common)                           │
│  ──────────────────────────────────────────────────     │
│                                                          │
│  When: Class does NOT implement interface               │
│                                                          │
│  @Service                                                │
│  public class UserService {                             │
│      @Transactional                                      │
│      public void saveUser(User user) { ... }            │
│  }                                                       │
│                                                          │
│  → Spring creates: UserService$$EnhancerBySpringCGLIB   │
│  → Subclass of UserService (extends)                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## Private Method Problem

### ❌ CRITICAL: @Transactional Does NOT Work on Private Methods!

```
┌──────────────────────────────────────────────────────────┐
│        WHY PRIVATE METHODS DON'T WORK                    │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  @Service                                                │
│  public class UserService {                             │
│                                                          │
│      @Transactional  ← IGNORED! Won't work!             │
│      private void saveUser(User user) {                 │
│          userRepository.save(user);                     │
│      }                                                   │
│  }                                                       │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                                          │
│  WHY IT DOESN'T WORK:                                   │
│                                                          │
│  1. Proxies can ONLY intercept PUBLIC methods           │
│                                                          │
│     ┌────────────────────────────────┐                 │
│     │  UserService$$Proxy            │                 │
│     │  (extends UserService)         │                 │
│     │                                │                 │
│     │  ❌ Can't override private!    │                 │
│     │  ✅ Can override public        │                 │
│     └────────────────────────────────┘                 │
│                                                          │
│  2. Private methods = Internal calls                    │
│                                                          │
│     Proxy doesn't even see the call!                    │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                                          │
│  RULE: @Transactional only works on:                   │
│  ✅ public methods                                       │
│  ❌ private methods                                      │
│  ❌ protected methods (not recommended)                 │
│  ❌ package-private methods (not recommended)           │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### Example: Private Method Failure

```java
@Service
@Slf4j
public class UserService {

    @Autowired
    private UserRepository userRepository;

    // ❌ BAD: This won't have transaction!
    @Transactional
    private void saveUserInternal(User user) {
        userRepository.save(user);
        // No transaction here! Changes commit immediately!
    }

    // This calls private method
    public void registerUser(User user) {
        saveUserInternal(user);  // ❌ No transaction applied!
    }
}
```

**What Actually Happens**:

```
┌──────────────────────────────────────────────────────────┐
│             PRIVATE METHOD CALL FLOW                     │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Controller calls:                                       │
│  userService.registerUser(user);                         │
│         │                                                │
│         ▼                                                │
│  ┌──────────────────┐                                   │
│  │  PROXY           │                                   │
│  │  registerUser()  │  ← Proxy intercepts public method│
│  └────────┬─────────┘                                   │
│           │                                              │
│           │ No @Transactional on registerUser()         │
│           │ Just passes through...                      │
│           │                                              │
│           ▼                                              │
│  ┌──────────────────┐                                   │
│  │  REAL OBJECT     │                                   │
│  │  registerUser()  │                                   │
│  │    │             │                                   │
│  │    └─ calls ──▶ saveUserInternal()  ← DIRECT CALL   │
│  │                  (bypasses proxy!)                   │
│  │                  NO TRANSACTION! ❌                  │
│  └──────────────────┘                                   │
│                                                          │
│  Result: saveUserInternal() executes WITHOUT transaction│
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## Common Pitfalls

### Pitfall 1: Self-Invocation (Internal Calls)

#### ❌ PROBLEM: Calling @Transactional method from same class

```java
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    // Public method WITHOUT @Transactional
    public void registerUser(User user) {
        validateUser(user);
        saveUser(user);  // ❌ Self-invocation - bypasses proxy!
    }

    // Has @Transactional but won't work when called internally!
    @Transactional
    public void saveUser(User user) {
        userRepository.save(user);
    }
}
```

**Flow Diagram**:

```
┌──────────────────────────────────────────────────────────┐
│           SELF-INVOCATION PROBLEM                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  External Call (Works ✅):                               │
│  ──────────────────────────────────────────────────     │
│                                                          │
│  controller.registerUser()                               │
│         │                                                │
│         ▼                                                │
│  ┌────────────┐  ✅ Goes through proxy                  │
│  │   PROXY    │  ✅ Transaction applied                 │
│  └──────┬─────┘                                         │
│         │                                                │
│         ▼                                                │
│  ┌────────────┐                                         │
│  │   REAL     │                                         │
│  │  saveUser()│  ✅ Executes in transaction            │
│  └────────────┘                                         │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                                          │
│  Internal Call (Fails ❌):                               │
│  ──────────────────────────────────────────────────     │
│                                                          │
│  controller.registerUser()                               │
│         │                                                │
│         ▼                                                │
│  ┌────────────┐                                         │
│  │   PROXY    │                                         │
│  └──────┬─────┘                                         │
│         │                                                │
│         ▼                                                │
│  ┌────────────────────────┐                            │
│  │   REAL                 │                            │
│  │  registerUser() {      │                            │
│  │    this.saveUser() ────┼─→ ❌ Direct call!          │
│  │  }                     │    Bypasses proxy!         │
│  │                        │    NO TRANSACTION!         │
│  │  saveUser() {          │                            │
│  │    // No transaction   │                            │
│  │  }                     │                            │
│  └────────────────────────┘                            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

#### ✅ SOLUTION 1: Separate into Different Services

```java
// Service 1: User Registration
@Service
public class UserRegistrationService {

    @Autowired
    private UserService userService;  // Inject OTHER service

    public void registerUser(User user) {
        validateUser(user);
        userService.saveUser(user);  // ✅ External call - works!
    }
}

// Service 2: User Operations
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    @Transactional
    public void saveUser(User user) {
        userRepository.save(user);  // ✅ Transaction works!
    }
}
```

#### ✅ SOLUTION 2: Self-Inject Using ApplicationContext

```java
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    @Autowired
    private ApplicationContext context;

    public void registerUser(User user) {
        validateUser(user);

        // Get proxy of self
        UserService self = context.getBean(UserService.class);
        self.saveUser(user);  // ✅ Goes through proxy!
    }

    @Transactional
    public void saveUser(User user) {
        userRepository.save(user);
    }
}
```

#### ✅ SOLUTION 3: Put @Transactional on Caller Method

```java
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    @Transactional  // ✅ Move annotation here
    public void registerUser(User user) {
        validateUser(user);
        saveUser(user);  // Now in same transaction
    }

    // Remove @Transactional from here
    public void saveUser(User user) {
        userRepository.save(user);
    }
}
```

### Pitfall 2: Protected/Package-Private Methods

```java
@Service
public class UserService {

    // ❌ NOT RECOMMENDED: Protected method
    @Transactional
    protected void saveUser(User user) {
        userRepository.save(user);
    }

    // ❌ NOT RECOMMENDED: Package-private method
    @Transactional
    void updateUser(User user) {
        userRepository.update(user);
    }
}
```

**Why not recommended**:
- Works with CGLIB proxy (by accident)
- Breaks if you switch to JDK proxy
- Not guaranteed by Spring documentation
- Confusing for team members

**Best Practice**: Always use `public` with `@Transactional`

---

### Pitfall 3: Final Methods

```java
@Service
public class UserService {

    // ❌ DOESN'T WORK: CGLIB can't override final methods
    @Transactional
    public final void saveUser(User user) {
        userRepository.save(user);
    }
}
```

**Why it fails**:
- CGLIB creates subclass (extends your class)
- Can't override `final` methods
- Transaction won't be applied

**Solution**: Remove `final` keyword

---

### Pitfall 4: Static Methods

```java
@Service
public class UserService {

    // ❌ DOESN'T WORK: Can't proxy static methods
    @Transactional
    public static void saveUser(User user) {
        userRepository.save(user);
    }
}
```

**Why it fails**:
- Proxies work on instance methods
- Static methods belong to class, not instance
- No proxy can intercept static calls

**Solution**: Use instance methods

## Interview Q&A

### Q1: How does @Transactional work internally?

**Answer**:
"Spring uses AOP (Aspect-Oriented Programming) with proxy pattern. When Spring detects `@Transactional` on a class or method, it creates a proxy wrapper.

When you call the method, you actually call the proxy first. The proxy:
1. Starts a transaction before the method
2. Calls the actual method
3. Commits on success or rolls back on exception

Spring uses either JDK Dynamic Proxy (if class implements interface) or CGLIB proxy (if class doesn't implement interface). The proxy intercepts public method calls and applies transaction management."

---

### Q2: Why doesn't @Transactional work on private methods?

**Answer**:
"Because proxies can only intercept calls that go through the proxy object. Private methods can't be overridden by subclasses, so CGLIB proxy can't intercept them.

Additionally, when you call a private method from within the same class (`this.privateMethod()`), it's a direct call that bypasses the proxy entirely. The proxy never sees the call, so transaction management isn't applied.

The rule is: `@Transactional` only works on public methods called from external classes."

---

### Q3: What is self-invocation problem?

**Answer**:
"Self-invocation happens when one method in a class calls another `@Transactional` method in the same class.

Example:
```java
public void methodA() {
    this.methodB();  // ❌ Self-invocation
}

@Transactional
public void methodB() {
    // Transaction doesn't work!
}
```

The problem is `this.methodB()` is a direct call, bypassing the proxy. The solution is to either:
1. Move methodB to a separate service and inject it
2. Put `@Transactional` on methodA instead
3. Self-inject using ApplicationContext (not recommended)"

---

### Q4: What's the difference between JDK proxy and CGLIB proxy?

**Answer**:

| Aspect | JDK Dynamic Proxy | CGLIB Proxy |
|--------|-------------------|-------------|
| **When used** | Class implements interface | No interface |
| **How it works** | Creates proxy implementing interface | Creates subclass extending your class |
| **Requirements** | Must have interface | Class can't be final |
| **Method access** | Only interface methods | All public methods |
| **Performance** | Slightly faster | Slightly slower |
| **Default** | Spring Boot 2.0+ prefers CGLIB | Most common |

"In modern Spring Boot, CGLIB is the default even when interfaces exist. You can see which proxy is used by checking the class name - CGLIB proxies have `$$EnhancerBySpringCGLIB` in the name."

---

### Q5: Can I use @Transactional on protected or package-private methods?

**Answer**:
"Technically, it works with CGLIB proxies because CGLIB creates a subclass and can override non-private methods. However, it's NOT recommended because:

1. **Not documented**: Spring documentation only guarantees public methods
2. **Brittle**: Breaks if you add interface (switches to JDK proxy)
3. **Confusing**: Team members expect transactions only on public methods

Best practice: Always use `public` with `@Transactional`."

---

### Q6: What happens if I put @Transactional on the class?

**Answer**:
"All public methods in that class become transactional:

```java
@Service
@Transactional  // ← Applied to all public methods
public class UserService {

    public void methodA() { }  // ✅ Transactional
    public void methodB() { }  // ✅ Transactional
    private void methodC() { } // ❌ Not transactional
}
```

You can override at method level:
```java
@Service
@Transactional
public class UserService {

    public void methodA() { }  // ✅ Transactional

    @Transactional(readOnly = true)
    public void methodB() { }  // ✅ Read-only transaction

    // Remove transaction
    @Transactional(propagation = Propagation.NOT_SUPPORTED)
    public void methodC() { }  // ❌ Not transactional
}
```"

## Best Practices

### ✅ DO's

```java
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    // ✅ DO: Use public methods
    @Transactional
    public void saveUser(User user) {
        userRepository.save(user);
    }

    // ✅ DO: Specify isolation level
    @Transactional(isolation = Isolation.READ_COMMITTED)
    public void updateUser(User user) {
        userRepository.update(user);
    }

    // ✅ DO: Mark read-only transactions
    @Transactional(readOnly = true)
    public User getUser(Long id) {
        return userRepository.findById(id);
    }

    // ✅ DO: Specify timeout
    @Transactional(timeout = 30)
    public void complexOperation() {
        // Long running operation
    }

    // ✅ DO: Specify rollback rules
    @Transactional(rollbackFor = Exception.class)
    public void riskyOperation() {
        // Rolls back on ANY exception
    }
}
```

### ❌ DON'Ts

```java
@Service
public class UserService {

    // ❌ DON'T: Private methods
    @Transactional
    private void saveUser(User user) { }

    // ❌ DON'T: Final methods
    @Transactional
    public final void saveUser(User user) { }

    // ❌ DON'T: Static methods
    @Transactional
    public static void saveUser(User user) { }

    // ❌ DON'T: Self-invocation
    public void registerUser(User user) {
        this.saveUser(user);  // Bypasses proxy
    }

    @Transactional
    public void saveUser(User user) { }

    // ❌ DON'T: Empty catch blocks (swallows rollback)
    @Transactional
    public void saveUser(User user) {
        try {
            userRepository.save(user);
        } catch (Exception e) {
            // ❌ Transaction won't rollback!
        }
    }
}
```

## Complete Working Example

```java
// ═══════════════════════════════════════════════════════════
//                    CORRECT USAGE
// ═══════════════════════════════════════════════════════════

@Service
@Slf4j
public class OrderService {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private PaymentService paymentService;  // Separate service

    @Autowired
    private NotificationService notificationService;

    // ✅ Correct: Public transactional method
    @Transactional(
        isolation = Isolation.READ_COMMITTED,
        timeout = 30,
        rollbackFor = Exception.class
    )
    public Order createOrder(OrderRequest request) {
        log.info("Creating order in transaction");

        // Save order
        Order order = new Order();
        order.setCustomerId(request.getCustomerId());
        order.setAmount(request.getAmount());
        orderRepository.save(order);

        // Call another service (✅ external call - works!)
        paymentService.processPayment(order);

        // Send notification (non-transactional)
        notificationService.sendOrderConfirmation(order);

        return order;
    }

    // ✅ Correct: Read-only transaction
    @Transactional(readOnly = true)
    public Order getOrder(Long orderId) {
        return orderRepository.findById(orderId)
            .orElseThrow(() -> new OrderNotFoundException(orderId));
    }
}

@Service
@Slf4j
public class PaymentService {

    @Autowired
    private PaymentRepository paymentRepository;

    // ✅ Correct: Requires existing transaction
    @Transactional(propagation = Propagation.MANDATORY)
    public void processPayment(Order order) {
        log.info("Processing payment in existing transaction");

        Payment payment = new Payment();
        payment.setOrderId(order.getId());
        payment.setAmount(order.getAmount());
        payment.setStatus(PaymentStatus.COMPLETED);

        paymentRepository.save(payment);
    }
}

@Service
@Slf4j
public class NotificationService {

    // ✅ Correct: Non-transactional (external service)
    public void sendOrderConfirmation(Order order) {
        log.info("Sending notification (no transaction needed)");
        // Send email/SMS
    }
}
```

## Key Takeaways for Interview

1. **@Transactional uses AOP with proxies** - Not magic, just a wrapper pattern
2. **Only public methods work** - Private/protected/static/final methods won't work
3. **Self-invocation breaks transactions** - `this.method()` bypasses proxy
4. **Solution: Separate services** - Move transactional methods to different services
5. **CGLIB is default** - Creates subclass, requires non-final methods
6. **Always test** - Use transaction logging to verify behavior
